1. Resumen Ejecutivo

El Dashboard de Ventas Farmacéuticas es una aplicación web diseñada para proporcionar visibilidad en tiempo real del desempeño de ventas de productos farmacéuticos a través de múltiples líneas comerciales. La plataforma integra datos de Odoo ERP y Google Sheets para ofrecer análisis detallados, seguimiento de metas y gestión de equipos de ventas.

🎯 Objetivo Principal: Centralizar y visualizar información crítica de ventas para facilitar la toma de decisiones estratégicas y operativas del equipo comercial.

2. Contexto del Negocio

2.1 Problema a Resolver

La empresa maneja múltiples líneas comerciales farmacéuticas (Petmedica, Agrovet, Pet Nutriscience, Avivet, Genvet, Interpet, entre otras) con equipos de vendedores distribuidos. Los desafíos principales incluyen:

  • Falta de visibilidad en tiempo real del avance de ventas vs. metas establecidas
  • Dificultad para analizar el rendimiento individual de vendedores por línea comercial
  • Datos dispersos entre diferentes sistemas (Odoo, hojas de cálculo)
  • Necesidad de identificar productos con próxima fecha de vencimiento
  • Seguimiento de productos nuevos (IPN - Introducción de Productos Nuevos)
  • Análisis del ciclo de vida de productos y formas farmacéuticas

2.2 Solución Propuesta

Una aplicación web centralizada que integre datos de ventas de Odoo ERP con la gestión de metas almacenada en Google Sheets, proporcionando dashboards interactivos con KPIs, gráficos analíticos y capacidades de exportación para análisis detallado.

5.4 Integraciones

Odoo ERP (XML-RPC)

  • Autenticación de usuarios contra la base de datos de Odoo
  • Extracción de líneas de venta con múltiples filtros
  • Obtención de lista de vendedores
  • Consulta de datos de productos, clientes, líneas comerciales

Google Sheets API

  • Almacenamiento de metas por línea comercial (pestañas mensuales)
  • Gestión de equipos de ventas (pestaña "Equipos")
  • Metas individuales de vendedores (estructura pivot por equipo/vendedor/mes)
  • Sincronización bidireccional (lectura y escritura)

5.5 Seguridad

  • Autenticación: Integrada con Odoo ERP
  • Lista Blanca: Control de acceso mediante allowed_users.json
  • Sesiones: Gestión de sesiones con Flask (SECRET_KEY en variables de entorno)
  • Control de Acceso Basado en Roles: Verificación de permisos de administrador en rutas sensibles
  • Variables de Entorno: Credenciales y configuración sensible en archivo .env
  • Manejo Global de Errores: 4 manejadores de errores implementados (@app.errorhandler para 403, 404, 500 y Exception) que previenen la exposición de información técnica sensible
  • Plantillas de Error Profesionales: Páginas de error personalizadas (error_403.html, error_404.html, error_500.html) con mensajes genéricos para usuarios
  • Ofuscación de Mensajes: Los errores técnicos se registran en logs con exc_info=True pero solo se muestran mensajes genéricos a los usuarios
  • Diferenciación DEBUG/PROD: Información técnica detallada solo visible en modo DEBUG, producción muestra mensajes sanitizados
  • Cumplimiento ISO 27001: Implementación de la Sección 5 (Manejo de Errores) con puntuación 91/100 y 85% de conformidad

7. Reglas de Negocio

7.1 Cálculo de Métricas

Métrica Fórmula
% Avance (Venta Total / Meta Total) × 100
% Avance IPN (Venta IPN / Meta IPN) × 100
Avance Diario Acumulado (% Avance Total / Día Actual del Mes)
Ritmo Diario Requerido (100% - % Avance Actual) / Días Laborables Restantes
Proyección Lineal (Venta Actual / Día Actual) × Días Totales del Mes
Faltante para Meta max(Meta Total - Venta Total, 0)

7.2 Filtros y Exclusiones

  • Ventas Internacionales: Se excluyen automáticamente las ventas marcadas como "VENTA INTERNACIONAL" o del canal "INTERNACIONAL"
  • Días Laborables: Se consideran de Lunes a Sábado (excluyendo Domingo)
  • Productos con Vencimiento: Se identifican productos con route_id en [18, 19] (rutas de vencimiento <6 meses)
  • Productos Nuevos (IPN): Se consideran productos con product_life_cycle = 'nuevo'

7.3 Limpieza de Datos

🔧 Normalización de Productos ATREVIA: Los productos de la línea ATREVIA se agrupan eliminando los indicadores de tamaño/presentación (MEDIUM, LARGE, SMALL, etc.) para consolidar las ventas por producto base.

Ejemplos:

  • ATREVIA ONE MEDIUM → ATREVIA ONE
  • ATREVIA XR LARGE → ATREVIA XR
  • ATREVIA TRIO CATS SPOT ON MEDIUM → ATREVIA TRIO CATS

7.4 Gestión de Ajustes

  • Las Notas de Crédito sin vendedor asignado se agrupan en una fila especial "Ajustes y Notas de Crédito"
  • Los ajustes se suman al total de ventas de la línea para mantener consistencia contable
  • Los vendedores con ventas negativas (solo notas de crédito) se excluyen de la tabla, pero su impacto se refleja en el total

8. Despliegue y Configuración

8.1 Requisitos del Sistema

  • Python 3.9 o superior
  • Conexión a Odoo ERP (URL, base de datos, credenciales)
  • Cuenta de servicio de Google Cloud con acceso a Google Sheets API
  • Servidor web compatible con Gunicorn (Linux/Unix)

8.2 Variables de Entorno (.env)

Variable Descripción
SECRET_KEY Clave secreta para sesiones de Flask
ODOO_URL URL del servidor Odoo
ODOO_DB Nombre de la base de datos Odoo
GOOGLE_SHEET_NAME Nombre de la hoja de Google Sheets para metas

8.3 Archivos de Configuración

  • credentials.json: Credenciales de la cuenta de servicio de Google Cloud
  • allowed_users.json: Lista blanca de usuarios autorizados
  • requirements.txt: Dependencias de Python

8.4 Estructura de Google Sheets

La hoja de cálculo debe contener las siguientes pestañas:

  • Equipos: Columnas equipo_id | vendedor_id | vendedor_nombre
  • Metas Vendedores: Estructura pivot equipo_id | vendedor_id | mes | meta | meta_ipn
  • Meses (YYYY-MM): Una pestaña por mes con columnas linea_id | meta | meta_ipn

8.5 Comando de Inicio

# Desarrollo
python app.py

# Producción (Gunicorn)
gunicorn -w 4 -b 0.0.0.0:5000 app:app

11.2 Campos de Datos de Odoo

Principales campos utilizados del modelo account.move.line:

  • invoice_date: Fecha de factura
  • move_name: Número de documento
  • partner_name: Nombre del cliente
  • invoice_user_id: Vendedor
  • name: Descripción del producto
  • quantity: Cantidad vendida
  • balance: Importe total
  • commercial_line_national_id: Línea comercial
  • product_life_cycle: Ciclo de vida del producto
  • route_id: Ruta de venta (para identificar vencimientos)
  • pharmaceutical_forms_id: Forma farmacéutica
  • sales_channel_id: Canal de ventas
